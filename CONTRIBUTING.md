# Contributing to VEX Projects

Thank you for your interest in contributing to the VEX ecosystem. We welcome contributions that improve the reliability, security, and interoperability of the protocol.

### Standards

*   **Hashing Parity**: All cryptographic implementations MUST maintain bit-for-bit parity with the v1.5.0 protocol specification (Merkle RFC 6962 and JCS RFC 8785).
*   **Safety First**: Security-critical code (signing, validation, entropy management) should be extensively tested and peer-reviewed.
*   **Documentation**: New features must include corresponding documentation and test vectors.

### Process

1.  **Issue First**: Please open an issue to discuss any major changes before submitting a pull request.
2.  **Pull Requests**:
    *   Target the `main` or `develop` branch.
    *   Ensure all tests pass (`npm test`, `pytest`, or `cargo test`).
    *   Include a clear description of the change and its impact.

### Code of Conduct

We are committed to providing a welcoming and inclusive environment for all contributors. Please be respectful and professional in all interactions.
